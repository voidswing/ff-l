# Agent Guide

## 핵심 원칙
- 코드는 짧고 읽기 쉽게 작성한다. 중복을 줄이고 함수/모듈 책임을 분리한다.
- 복잡한 로직에는 의도를 설명하는 짧은 주석을 남긴다. 코드가 자명하면 주석은 생략한다.
- 기능 추가/버그 수정 시 테스트를 함께 작성·수정하고, 최소 1개 이상의 회귀 테스트를 포함한다.
- 커밋 전 관련 테스트를 실행해 실패를 해결한 뒤 반영한다.
- 작업 시작 전에 `docs/`를 확인해 재사용 가능한 규칙/결정을 우선 적용한다.
- 반복 활용 가능성이 높은 결정, 운영 절차, 트러블슈팅은 `docs/*.md`로 문서화하고 커밋/PR에 경로를 남긴다.
